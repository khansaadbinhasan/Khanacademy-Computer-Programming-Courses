I remembered from my childhood a software called typing master. It was an awesome piece of software and i downloaded it and found it to be very helpful. I spent half an hour a day eating through the practice sessions. And after a month I guess i completed all the tutorials and then began the paid version which i didnt took. Also my typing was good enough for me at that point and i had other concerns like programming to think of. So i stopped typing practice. Then After about six months I got my own laptop and decided that I am going to improve my typing.

At that point i came across this website called keybr and instantly fell in love with it. I spent next three months and about 30 hrs on that website to become a better typist. Starting off my typing speed was only 25wpm but with time it increased and by the time i left my top speed was touching 75 wpm and average speed well over 60wpm. The initial goal i set was 50wpm So I was very much satisfied with the result. I would highly recommend both typing-master and keybr if you want to learn typing. And after you are done with them you can use typeracer and ten-fast-fingers.

I personally like typeracer alot because it gives you a platform to show to others how good your typing is and by going into a competition you can be certain of your position around the world. Also it gives you a feeling of pride. To this day typing remains the only thing i am seriously proud of. So Heres the order of softwares/websites I would recommend to learn typing:

  1. Typing Master
  2. Keybr
  3. Type-racer

Getting Started with Linux

While learning programming I heard a lot of bad things about windows. Many of my friends said that if you want to become a real programmer abandon windows because it will take you nowhere. I read many articles suggesting to switch away from windows as soon as possible. At that time i didn't understand a thing about operating systems. But since Everyone was saying those things about windows I was startled but curious so I decided to give linux a try. So when I had my own laptop i dual booted that with ubuntu.

Initially it wasn't that interesting and most of the time I would go back to using windows and within the first three months i rarely used it except for programming purposes but as time went by windows started to disappoint me more and more. The main problem was of updates. Since I had only limited amount of mobile data i wanted to save it for educational purposes instead of wasting it on some stupid updates of which i wasnt even aware of. I tried to stop them but there was no choice. Also my windows was hanging alot and the computer was getting slower and slower everyday. Till then I was using windows only for gaming purposes so one day I decided to do away with windows and uninstalled it. Its been a year now and there is not a single thing that ubuntu cant do that windows can except maybe gaming and other graphics stuff.

Now using linux operating system has its own sets of challenges because most of us were born and raised using windows. So its difficult for us to get used to any other user-interface. But Ubuntu makes it very easy. I have Ubuntu 16.04 for almost a year now and I dont find any problem interacting with it. Its alot faster than windows and needs less resources. Even very old computers can use Ubuntu. My decision to use Ubuntu was also an ethical one. I was tired of windows interfering with everything I felt that i was being controlled and the notion that they were selling my information to others really made me angry. Then i read about the open source movement and free softwares. And linux was at the head of that. So i had to use linux any way.

Now Linux is an open source operating system developed by Linus Torvalds. This means that all its source code is available free of cost and you can alter it as you please. This idea is very empowering because since all source code is available you can check for backdoors and remove them if there are any. Its also free of cost and puts an end to paid closed source softwares which dont even respect your privacy. Linux Torvalds only developed the 'Linux Kernel' which is basically the part that interacts with the hardware. But normal users dont use kernel directly because you need to know a great deal about computers before interacting with kernels so instead there are layer over the kernel to help users interact in an easier way. After all the layers there are GUI-graphical user interface that most user use in windows i.e, by using mouse and seeing icons we interact with them and then there is CLI-command line interface which is faster than GUI easier to navigate and takes less time to perform any action and also needs less resources but it is very hard to learn. Windows users interact with GUI mode all their life without understanding whats going on under the hood. But linux is different

As mentioned above The user needs a user-interface to interact with the kernel, These are provided by many companies and all the different user-interfaces are called distributions or distros. One of the most popular distro of linux is ubuntu. Because it provides a good GUI experience almost similart to windows so it is easy to make a transition to linux from windows. Also Ubuntu forums are very vibrant and you can get answer to any question within minutes.But if you want to use linux you must use CLI because thats what makes linux stand apart. There is a program called terminal in ubuntu that helps us in this. Terminal seems boring and intimidating at first but with enough practice it becomes better and better.

The main benefit of using terminal is that you can download any software just by using a single command. Instead of going to the softwares website donwloading it, unzipping it , installing it , clicking OK a dozen time you can just type `sudo apt-get install software-name`
The above command works in most cases and saves us from a ton of work we will have to do in windows. If you want to install some other software you can go to its website and simply copy paste a bunch of commands to install it. Also since linux is open-source if you find any problem with any of the functioning you can report it and it will be fixed in less than a week. If you know enough programming you can customize it the way you want. This is a very desirable property because different industries have different needs for computers. For example weather forecasting computers may need to run only weather related softwares so they can customize linux to suit their needs and so can Internet servers. But with windows you can not do this without paying big bucks to company to do that for you or downloading costly softwares.

Coming back to terminal, Its very hard in the beginning to learn the commands because there are so many of them I faced the same problem but then i came across these awesome tutorials I did only the first five or so tutorials but they were quite helpful in getting started. I then started using them and finding more commands on stack-overflow as and when needed soon i felt like a wizard using the terminal. I still did some more courses though newboston linux tutorials and then this codeacademy linux tutorials. These courses are more than enough to give you a good experience with command line. While doing this I kept learning various keyboard shortcuts. All this helped me learning how to use linux and now i am pretty comfortable in using ubuntu and command line.

Getting Started with Git , Shell Scripting , Sublime , Vim

Vim is a text-editor that means you can edit text files in it. Vim is one of the oldest text-editors but is still very popular and most seasoned programmers prefer it over other alternatives like sublime , atom , notepad etc. One of the main reason for that it is command line based so you can do all your work in terminal without going through the trouble of switching between tabs all the time, Also it is very powerful because you can do all your work without having to touch mouse, this gives you immense speed improvement. You dont have to move your hand anywhere except on the keyboard. But like all other linux things it has a steep learning curve, When i started learning vim i almost gave up because i liked sublime more as it has a nice look and feel and i knew all the shortcuts in it. Its autocompletion was awesome and suggestions were good too. But the thing i liked the most was the keyboard shortcuts. There were many of them but they were all very useful. Nonetheless i continued learning vim because it seemed like a good thing to invest time on. I learned from vim-tutor which are official tutorials for learning vim. I guess they are good enough if you want to learn vim fast. You can complete the whole tutorials in about an hour and then you need to practice alot to get better grip on vim. The main problem i faced and still facing is the autocompletion, there are many plugins for that like you-complete-me but i am unable to install them and the reason is beyond my reach. But i still use vim ocassionally and its very satisfyings.

Git is a version control system. It is used when we are doing programming or working on a software. While building softwares we keep changing our code maybe because we are fixing some bug or tweaking with something, now this may result in catastrophic problems because you are continously changing things which may break the things that were otherwise working already, To avoid this we have version control systems, What they do is that they save the state of the system at some point in time which maybe specified by user while doing some changes like in git you need to execute a `commit` command to tell git to store the information of code at current state. This is very usefule because after saving you can make any number of changes you want and if they work fine, then its good else you can undo the changes and come back to square one. Git is also very useful when more than one person is working on a software which is the case usually because softwares are big and you need many people to work on them, Now they need to collaborate in some way that is through git, For example if you have been asked to develop some part of the code and you do that now you need to patch that up with the main code. So what you do is that you make a patch and commit it to the parent branch, Then your project manager can see that its working correctly or not, Then he/she can tell you to make some changes or accept your patch as is. This reduces the chance of any problem that may arise and even if you have any problem you can always go back.

I wanted to learn git out of curiousity. And these tutorials were very useful in that: newboston and Learning git is more or less like learning about the command line, Its like an added feature of command line, So if you know command line and directory structure then it would be very easy for you to learn git.

Shell scripting is used for automating the computer, you can make a shell script execute it and sit back to see what it does and enjoy the show, it is one of the coolest thing i found on linux and i instantly fell in love with it. Its basically a bunch of commands in a file with .sh extension Some people also call it a different language. I found these tutorials very useful Shell Scripting Tutorials
